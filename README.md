# Immersive Reader — Feature Demo

A working prototype of the Immersive Reader feature, built on the project's existing frontend stack.

**[▶ View Live Demo](https://YOUR-USERNAME.github.io/immersive-reader-demo)**

---

## What this demos

| Feature | Description |
|---|---|
| 🔊 **TTS Read-aloud** | Browser speech synthesis reads the text aloud |
| 🟡 **Word-sync highlight** | Each word highlights in real-time as it's spoken (via `onboundary` event) |
| 👁 **Bionic Reading** | Bolds the first half of each word to guide the eye |
| ⚡ **Speed control** | Preset speeds (0.8x–2.5x) + fine stepper + auto-speed mode |
| 🖱 **Click to seek** | Click any word to jump playback to that position |

---

## Stack used

This demo is built with the **same stack as the main project** — no new dependencies:

| Tool | Role |
|---|---|
| **Tailwind CSS** | All layout and styling (pre-built via CLI — no browser compiler) |
| **Alpine.js** | All interactivity via `x-data="immersiveReader()"` |
| **Web Speech API** | Browser-native TTS — no external service needed |

The Alpine component is structured to drop straight into `resources/js/` and register as `Alpine.data('immersiveReader', immersiveReader)`.

Book content is hardcoded for this demo — in production it comes from Laravel as `@json($book->paragraphs)`.

## Regenerate CSS after changing classes

```bash
npm install
npx tailwindcss -i input.css -o style.css --content index.html
```
